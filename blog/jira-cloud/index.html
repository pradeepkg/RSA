Jira Cloud

October 13, 2023 · 41 min read
Pradeep Kadambar
Creator
Jira Cloud is a popular project management tool that helps teams plan, track, and manage their work efficiently. It provides a centralized platform for creating tasks, organizing workflows, assigning responsibilities, and tracking progress. With features like customizable boards, issue tracking, real-time collaboration, and integrations with other tools, Jira Cloud streamlines project management processes, improves team communication, and facilitates agile development methodologies, ultimately enhancing productivity and delivering high-quality results. 

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ✔️ | Entitlements ❌

Users - Users are individuals who have an account in Jira and can log in to access and work on projects.

Groups - Groups are collections of users. By assigning permissions to groups, you can easily manage access for multiple users simultaneously.

Roles - Roles represent a set of responsibilities or functions within a project. Examples include Project Lead, Developer, Tester, etc. Roles can be assigned to users or groups within a specific project.

Permissions - Permissions control what actions or operations users can perform within Jira. These permissions can be assigned at different levels, such as global (system-wide), project, or issue level. They define who can create, edit, view, transition issues, manage workflows, administer projects, and more.

Account Collector​

This section explains the process for configuring an account collector for the Jira Cloud application. We will use the RSA G&L Generic REST collector 1 for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Jira Cloud.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details. The endpoint uses a basic authentication using the API token. Refer to API Setup.

  6. Click Next. Select the data types that the collector will be collecting from Jira Cloud

  7. Click Next. On the Account Data page, enter the following configurations.



Request
MethodGET
URL/rest/api/3/users
Request Headers
Acceptapplication/json
Response Mappings
AccountId$.*.accountId
Email$.*.emailAddress
isActive$.*.active
DisplayName$.*.displayName
  1. Click Next. On the Group Data page, enter the following configurations.




note

The API doesn't return group descriptions.

Request
MethodGET
URL/api/v4/groups
Request Headers
Acceptapplication/json
Response Mappings
GroupId$.groups.*.name
Id$.groups.*.groupId
  1. Click Next. On the Account Group Membership page, enter the following configurations.




danger

Currently, this configuration is unsupported since G&L REST collectors do not replace placeholder tokens in query parameter. Jira does not support a single API to retrieve group members.

Request
MethodGET
URL/rest/api/3/group/member
Query Parameters
groupId${Id}
Request Headers
Acceptapplication/json
Response Mappings
Account Identifier$.*.username
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account.

💡 Info: This value MUST always be mapped to the Account Name for any generic REST collector.

  1. Click Finish. Use the Test function to make sure the configurations are accurate.

Entitlement Collector​

This section explains the process for configuring an entitlement collector to collect roles from Jira Cloud application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on Jira Cloud.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector.
  6. Click Next. Select the data types that the collector will be collecting from Jira Cloud.

  7. Click Next. On the App Role Data page, enter the following configurations.



Request
MethodGET
URL/rest/api/3/role
Request Headers
Acceptapplication/json
Response Mappings
ApplicationRoleId$.*.name
Id$.*.id
Description$.*.description
  1. Click Next. On the App-Role Account Membership collection page, enter the following configurations.




Request
MethodGET
URL/rest/api/3/role
Request Headers
Acceptapplication/json
Response Mappings
Application Role Id$.*.name
Account Identifier$.*.actors.*.actorUser.accountId
  1. Click Next. On the App-Role Group Membership Data page, enter the following configurations.




Request
MethodGET
URL/rest/api/3/role
Request Headers
Acceptapplication/json
Response Mappings
Application Role Id$.*.name
Account Identifier$.*.actors.*.actorGroup.groupId
  1. Click Next. In the Map Collector Attributes to App Role Attributes, map the collected account attributes.
  2. Click Next. In the Account Evaluation, map the account collector to account value.

  3. Click Next. In the Group Evaluation, map the account collector to group value.

  4. Click Next and then click Finish. Use the Test function to make sure the configurations are accurate

Jira Cloud: Setup​

API Token​

This section explains the process of acquiring the authentication data required for collectors from Jira Cloud. Refer to Basic authentication section in the API guide 1 for more information.

Steps

  1. Login to Jira Cloud Manage Account page https://id.atlassian.com/manage-profile/security
  2. Click on Create and manage API tokens under API token section

  3. Click on Create API token.
  4. Enter the token label and click Create.

  5. Copy the API token and save it for use within collector configuration.

  6. Click Close.
