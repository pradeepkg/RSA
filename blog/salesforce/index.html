In Salesforce, the relationship between users, roles, profiles, and permission sets is critical for managing access and permissions within the platform. These elements work together to control what users can do and see in the Salesforce environment.

Users

  • Users are individuals who interact with Salesforce. They can be employees, customers, partners, or any other stakeholders.
  • Each user is associated with a Salesforce account, and they have a unique username and password to access the system.

Groups

  • Groups in Salesforce are collections of users, roles, or other groups.
  • Groups can be used to grant access to records, share files, collaborate on projects, or assign ownership of records. Types of Groups:
    • Regular Groups - Standard groups that can contain users, roles, or other groups.
    • Queue Groups - Used for assigning ownership of records in certain Salesforce objects, such as cases or leads, to a group of users.
    • Public Groups - Groups accessible to all users in the organization.
    • Private Groups - Groups with restricted access, typically only accessible to members explicitly added to the group.

Roles

  • Roles define the hierarchy within your organization. They represent a way to structure your organization's data and specify who can access or see records owned by users lower in the hierarchy.
  • Roles are used to determine the level of visibility users have to records based on their position in the organization's hierarchy.

Profiles

  • Profiles are a collection of settings and permissions that determine what a user can do within Salesforce. They act as templates for user access.
  • A profile includes settings for object and field-level security, record types, page layouts, and more.
  • Each user is assigned to a specific profile when they are created. This profile dictates the basic level of access and functionality that user has.

Permission Sets

  • Permission sets are used to extend the permissions granted by profiles. They allow administrators to grant additional permissions and access to specific users without changing their profiles.
  • Permission sets are often used to provide extra capabilities to users who need them, such as access to custom objects, fields, or specific administrative tasks.
  • Users can be assigned multiple permission sets to combine permissions from different sources.

Here's how these elements are related in Salesforce:

  • Users are associated with profiles initially. The profile determines the baseline access and permissions for each user.
  • Roles define the hierarchy within the organization and influence record-level access for users based on their position in the hierarchy.
  • Permission sets can be assigned to users to grant them additional permissions beyond what their profiles allow. Permission sets are additive, meaning they add to the permissions granted by the user's profile.
  • The combination of profiles, roles, and permission sets determines the overall access and permissions for each user in the organization.

Profiles‚Äã

This section explains the process for configuring an entitlement collector to collect profiles and corresponding user assignments from Salesforce application. We will use the RSA G&L Generic REST collector üìò for this purpose.

Profiles
Profile

Profile is a metadata object that defines the permissions and settings that control what users can do within the organization. Each user in Salesforce is assigned a profile that determines their access level, object permissions, and user interface settings. Profiles are predefined sets of permissions that can be assigned to multiple users.

  • Profiles control permissions such as object permissions (e.g., create, read, edit, delete), field permissions, Apex class and Visualforce page access, user interface settings (e.g., tabs, apps, page layouts), and more.
  • Profiles are managed in Setup > Administration Settings > Profiles.
UserProfile

UserProfile is an object in the Salesforce schema that represents the individual user settings and configurations for a specific user. It contains information such as the user's language, locale, time zone, email preferences, and other personal settings.

  • UserProfile records are automatically created for each user in Salesforce and cannot be manually created or edited.
  • UserProfile records are used to store user-specific settings and configurations that are not controlled by profiles, such as language, locale, time zone, email preferences, etc.
  • UserProfile records can be accessed via the Salesforce Object Query Language (SOQL) queries but are typically managed programmatically rather than through the Salesforce UI.

  3. On the Connection Details page, configure the following.
tip
