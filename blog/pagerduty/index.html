PagerDuty

August 19, 2023 · 49 min read
Pradeep Kadambar
Creator
PagerDuty is a leading digital operations management platform that empowers teams to proactively manage and respond to incidents in real time. By providing actionable insights, automation, and collaboration tools, PagerDuty ensures business continuity and exceptional customer experiences. It centralizes alerts, streamlines incident resolution, and enables teams to work seamlessly across the entire organization, minimizing downtime and optimizing operational efficiency.

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ✔️ | Entitlements ❌

Teams are a way to organize users and objects in PagerDuty. They can be used to control who has access to different objects, such as escalation policies, users, schedules, and services. Teams can also be used to customize the PagerDuty web app by filtering for information relevant to the team's associated objects.

User roles define the level of access that a user has to PagerDuty objects.

RoleDescription
adminHas full access to all objects and configurations in the account. Can create, update, and delete users, teams, schedules, escalation policies, services, and integrations. Can also manage user permissions.
limited_userCan view objects and configurations, but cannot make any modifications.
observerCan view objects, but cannot make any modifications.
ownerHas full access to objects and configurations within their team. Can create, update, and delete users, schedules, escalation policies, and services. Can also manage user permissions within their team.
read_only_userCan view objects, but cannot make any modifications.
restricted_accessCan only view incidents and alerts that they are directly involved in.
read_only_limited_userCan only view objects that they have been granted access to.
userCan view objects and configurations, but cannot make any modifications.

Account Collector​

This section explains the process for configuring an account collector for the PagerDuty application. We will use the RSA G&L Generic REST collector 1 for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on PagerDuty.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details.

  6. Click Next. Select the data types that the collector will be collecting from PagerDuty

  7. Click Next. On the Account Data page, enter the following configurations.



Request
MethodGET
URL/users
Request Headers
Acceptapplication/json
AuthorizationToken token=${AuthenticationToken}
Response Mappings
AccountId$.users.*.email
Id$.users.*.id
Type$.users.*.type
  1. Click Next. On the Group Data page, enter the following configurations.




Request
MethodGET
URL/groups
Query Parameters
fieldsid,name
Request Headers
Acceptapplication/json
AuthorizationToken token=${AuthenticationToken}
Response Mappings
GroupId$.teams.*.name
Id$.teams.*.id
Description$.teams.*.description
  1. Click Next. On the Account Group Membership page, enter the following configurations.




Request
MethodGET
URL/groups/${Id}/users
Request Headers
Acceptapplication/json
AuthorizationToken token=${AuthenticationToken}
Response Mappings
Account Identifier$.members.*.user.id
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account.

💡 Info: This value MUST always be mapped to the Account Name for any generic REST collector.

  1. Click Finish. Use the Test function to make sure the configurations are accurate.

Entitlement Collector​

This section explains the process for configuring an entitlement collector to collect user roles and assignments from the PagerDuty application. We will use the RSA G&L Generic REST collector for this purpose.

info

Since PagerDuty does not provide an API for listing user roles, we will use a custom web service for the EDC. Refer to Custom Web Service : User Roles for more information.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on PagerDuty.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector. 💡 Note that the Use Base URL for all API(s) will be unchecked.
  6. Click Next. Select the data types that the collector will be collecting from PagerDuty.

  7. Click Next. On the App Role Data page, enter the following configurations.


Request
MethodGET
URLhttps://instance.securid.com/aveksa/custom/external_jsp/PagerDutyUserRolesv1.jsp
Request Headers
Acceptapplication/json
Response Mappings
ApplicationRoleId$.roles.*.role
Description$.roles.*.description
  1. Click Next. On the App-Role Account Membership collection page, enter the following configurations.




Request
MethodGET
URLhttps://api.pagerduty.com/users
Request Headers
Acceptapplication/json
AuthorizationToken token=${AuthenticationToken}
Response Mappings
Application Role Id$.users.*.role
Account Identifier$.users.*.id
  1. Click Next. In the Map Collector Attributes to App Role Attributes, map the collected account attributes.

  2. Click Next. In the Account Evaluation, map the account collector to account value.

  3. Click Next and then click Finish. Use the Test function to make sure the configurations are accurate.

Application Setup​

This section explains the process of setting up an API access key in PagerDuty for use within the collections process.

Steps

  1. Login to PagerDuty console https://instance.pagerduty.com.
  2. Under Integrations, click on API Access Keys.

  3. Click on Create new API key.
  4. Provide a meaningful name. Since we are going to use this key for collections, check the Read-only API Key option.
  5. Click on Create Key.
  6. Copy the Secret and click Done.

Custom Web Service​

Site Roles​

Since PagerDuty does not provide an API for listing site roles, we will use a custom JSP that will publish the site roles.

Steps

  1. Open a text editor and save the contents below as PagerDutyUserRolesv1.jsp.
<%@ page language="java" contentType="application/json" %>
<%
    String jsonData = "{"
        + "\"roles\": ["
        + "{\"role\":\"admin\",\"description\":\"Has full access to all objects and configurations in the account. Can create, update, and delete users, teams, schedules, escalation policies, services, and integrations. Can also manage user permissions.\"},"
        + "{\"role\":\"limited_user\",\"description\":\"Can view objects and configurations, but cannot make any modifications.\"},"
        + "{\"role\":\"observer\",\"description\":\"Can view objects, but cannot make any modifications.\"},"
        + "{\"role\":\"owner\",\"description\":\"Has full access to objects and configurations within their team. Can create, update, and delete users, schedules, escalation policies, and services. Can also manage user permissions within their team.\"},"
        + "{\"role\":\"read_only_user\",\"description\":\"Can view objects, but cannot make any modifications.\"},"
        + "{\"role\":\"restricted_access\",\"description\":\"Can only view incidents and alerts that they are directly involved in.\"},"
        + "{\"role\":\"read_only_limited_user\",\"description\":\"Can only view objects that they have been granted access to.\"},"
        + "{\"role\":\"user\",\"description\":\"Can view objects and configurations, but cannot make any modifications.\"}"
        + "]"
        + "}";
%>

<%
    response.setContentType("application/json");
    out.print(jsonData);
%>
  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Admin > User Interface.
  3. Click on Files.
  4. Select External JSP Pages and click on Upload.
  5. Select the PagerDutyUserRolesv1.jsp and click OK.
  6. Verify by accessing this URL in the browser https://instance.securid.com/aveksa/custom/external_jsp/PagerDutyUserRolesv1.jsp
