Amazon AWS

October 9, 2023 · 24 min read
Pradeep Kadambar
Creator
AWS Identity and Access Management (IAM) enables you to manage access to AWS services and resources securely. Using IAM, you can create and manage AWS users and groups, and use permissions to allow and deny their access to AWS resources.

Identities ❌ | Accounts ✔️ | Groups ❌ | App Roles ✔️ | Entitlements ❌

Application​

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Application.
  3. Click on Create Application and select Other Application.
  4. Enter the details as shown below.
  5. Click Finish.

Account Collector​

This section explains the process for configuring an account collector for the Amazon AWS IAM application. We will use the RSA G&L out of the box collector for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Amazon AWS.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details.Enter the proxy details if applicable to your environment. Refer to AWS Setup (Security Credentials) for more information on creating the API keys.

    .
  6. Click Next. The only available account attribute that can be collected is the AccountName.

  7. Click Next. On the mapping screen, add the AccountName as the User Reference field.

  8. Click Next. In the User Resolution Rules, map the user to account

  9. Click Next. In the Member Account Resolution Rules, map the target collector for group members.

  10. Click Finish. Use the Test function to make sure the configurations are accurate.

  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector.

AWS Setup (Security Credentials)​

This section explains the setup of security credentials for the service account that will be used with RSA G&L ADC and EDC.

Steps

  1. Login to AWS Identity and Access Management (IAM) console https://console.aws.amazon.com/iam/home.
  2. Expand Access Management and click on Policies.

  3. Click on Create Policy.
  4. Click on JSON tab and paste the below policy.
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "VisualEditor0",
            "Effect": "Allow",
            "Action": [
                "iam:ListGroupsForUser",
                "iam:ListUsers",
                "iam:ListGroups"
            ],
            "Resource": "*"
        }
    ]
}
  1. Click on Review Policy.

  2. Click Create Policy.
  3. Repeat Steps 3 through 6 to create another policy (RSA-IGL-EDC-Policy) for EDC using the below policy definition.
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "VisualEditor0",
            "Effect": "Allow",
            "Action": [
                "iam:ListGroupPolicies",
                "iam:ListAttachedRolePolicies",
                "iam:ListRoles",
                "iam:ListUserPolicies",
                "iam:ListRolePolicies"
            ],
            "Resource": "*"
        }
    ]
}
  1. Expand Access Management and click on Users.

  2. Click on Add User.
  3. Provide a user name for the service account and select Programmatic Access.

  4. Click Next: Permissions
  5. Under Set Permissions, click on Attach existing policies directly
  6. Select the two (2) policies created earlier for ADC & EDC.
  7. Click Next:Tags.
  8. Click Next :Review
  9. Click Create User.
  10. Click on the newly created user and navigate to the Security Credentials tab.
  11. Click Create Access Key.

  12. Click on Download.csv file to save the Access Key and Secret Key. We will need this file during the ADC and EDC configurations in RSA G&L.
