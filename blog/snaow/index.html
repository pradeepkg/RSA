ServiceNow

October 8, 2023 · 32 min read
Pradeep Kadambar
Creator
ServiceNow is a cloud-based platform offering IT service management, enabling organizations to streamline workflows, automate tasks, and enhance service delivery. It encompasses various modules for IT, HR, customer service, and more, promoting efficiency and collaboration within enterprises.

Application​

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Application.
  3. Click on Create Application and select Other Application.
  4. Enter the details as shown below.
  5. Click Finish.

Account Collector​

This section explains the process for configuring an account collector for the ServiceNow application. We will use the RSA G&L Generic REST collector 1 for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on ServiceNow.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details.



  6. If authentication type 2 is Basic, enter the credentials as shown below.

  7. Click Next. Select the data types that the collector will be collecting from ServiceNow

  8. Click Next. On the Account Data page, enter the following configurations.



Request
MethodGET
URL/api/now/table/sys_user
Query Parameters
sysparm_fieldssys_id,name,employee_number,user_name,email,first_name,last_name,last_login_time,active
Request Headers
Acceptapplication/json
Response Mappings
AccountId$.result[*].user_name
EmployeeNumber$.result[*].employee_number
IsActive$.result[*].active
Email$.result[*].email
LastLoginTime$.result[*].last_login_time
SysID$.result[*].sys_id
  1. Click Next. On the Group Data page, enter the following configurations.




Request
MethodGET
URL/api/now/table/sys_user_group
Query Parameters
sysparm_fieldssys_id,name,descraiption,active,sys_updated_by
sysparm_queryactive=true
Request Headers
Acceptapplication/json
Response Mappings
GroupId$.result[*].name
SysID$.result[*].sys_id
Description$.result[*].description
Owner$.result[*].sys_updated_by
  1. Click Next. On the Account Group Membership page, enter the following configurations.



Request
MethodGET
URL/api/now/table/sys_user_grmember
Query Parameters
sysparm_fieldsuser,group
Request Headers
Acceptapplication/json
Response Mappings
Account Identifier$.result[*].user.value
Group Identifier$.result[*].group.value
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account.

💡 Info: This value MUST always be mapped to the Account Name for any generic REST collector.

  1. Click Next. In the Group Owner Resolution Rules, map the owner to user attribute.

  2. Click Finish. Use the Test function to make sure the configurations are accurate.

Entitlement Collector​

This section explains the process for configuring an entitlement collector to collect admin roles from ServiceNow application. We will use the RSA G&L Generic REST collector 1 for this purpose.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on ServiceNow.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector.
  6. Click Next. Select the data types that the collector will be collecting from ServiceNow.

  7. Click Next. On the App Role Data page, enter the following configurations.




Request
MethodGET
URL/api/now/table/sys_user_role
Query Parameters
sysparm_fieldssys_id,sys_updated_by,name,description
sysparm_exclude_reference_linktrue
sysparm_queryORDERBYsys_id
Request Headers
Acceptapplication/json
Response Mappings
ApplicationRoleId$.result[*].name
RoleSysID$.result[*].sys_id
Description$.result[*].description
Owner$.result[*].sys_updated_by
  1. Click Next. On the App-Role Account Membership collection page, enter the following configurations.




Request
MethodGET
URL/api/now/table/sys_user_has_role
Query Parameters
sysparm_fieldsrole,user
sysparm_exclude_reference_linktrue
sysparm_queryroleISNOTEMPTY^userISNOTEMPTY^ORDERBYsys_id
Request Headers
Acceptapplication/json
Response Mappings
Application Role Id$.result[*].role
Account Identifier$.result[*].user
  1. Click Next. In the Map Collector Attributes to App Role Attributes, map the collected account attributes.

  2. Click Next. In the Account Evaluation, map the account collector to account value.

  3. Click Next and then click Finish. Use the Test function to make sure the configurations are accurate.
