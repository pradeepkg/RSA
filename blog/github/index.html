GitHub Enterprise Cloud

October 12, 2023 · 22 min read
Pradeep Kadambar
Creator
GitHub Enterprise Cloud is a fully managed and secure cloud-based solution that allows organizations to host their repositories and collaborate on code without the need for infrastructure management. It provides access to all the features of GitHub, including version control and issue tracking, while offering the convenience and scalability of cloud hosting. With GitHub Enterprise Cloud, organizations can focus on their development workflows while ensuring security and reliability.

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ❌ | Entitlements ❌

Members - Members of an organization can have various roles and permissions assigned to them. The specific permissions granted to members depend on their roles within the organization. Members can be added to teams and granted access to repositories based on the team permissions.

Teams - Teams are groups of organization members that can be assigned specific roles and permissions. You can create teams within an organization and define their access levels to repositories. Teams allow you to manage permissions for multiple members at once.

Account Collector​

This section explains the process for configuring an account collector for the GitHub Enterprise Cloud application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on GitHub Enterprise Cloud.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details. The endpoint uses a static token for authentication.

    .
  6. Click Next. Select the data types that the collector will be collecting from GitHub Enterprise Cloud

  7. Click Next. On the Account Data page, enter the following configurations.



Request
MethodGET
URL/orgs/ORG/members (Replace ORG with your organization identifier)
Request Headers
Acceptapplication/json
Response Mappings
AccountId$.*.login
Id$.*.id
State$.*.url
  1. Click Next. On the Group Data page, enter the following configurations.




Request
MethodGET
URL/orgs/ORG/teams
Request Headers
Acceptapplication/json
Response Mappings
GroupId$.*.name
Id$.*.id
Description$.*.description
SLUG$.*.slug
  1. Click Next. On the Account Group Membership page, enter the following configurations.




Request
MethodGET
URL/orgs/ORG/teams/${SLUG}/members
Request Headers
Acceptapplication/json
Response Mappings
AccountId$.*.login
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the collected group attributes.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account attribute.

  6. Click Finish. Use the Test function to make sure the configurations are accurate.
