Okta

September 9, 2023 · 62 min read
Pradeep Kadambar
Creator
Okta is a cloud-based IAM platform that offers secure user authentication, SSO, and authorization. It simplifies identity management, access control, and provisioning for organizations. With Okta, businesses can enhance security, streamline user experiences, and enable seamless access to applications and data. It supports multifactor authentication and integrations with various systems, enabling secure digital transformation and centralized identity management.

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ✔️ | Entitlements ❌

Users - Okta allows you to create and manage user accounts, including attributes like name, email, and profile information. Users can be assigned to various groups and roles.

Groups - Okta enables the creation of groups to organize users based on common attributes or roles. Groups help streamline user management and permissions assignment.

Group Members - Okta allows you to add users as members of specific groups. This helps control access to resources and streamline user provisioning and management.

Roles - Okta supports the definition and assignment of roles. Roles represent a collection of permissions or access privileges that can be granted to users or groups.

Role Members - Okta enables the assignment of users or groups as members of specific roles. This allows for fine-grained access control and authorization based on role-based permissions.

Account Status Filter Options

For use with filter query parameter in the API call below.

Account StatusDescription
ACTIVEThis status indicates that the user account is currently active and can access resources within the system.
PROVISIONEDUsers with this status have been provisioned into the system but may not have completed their activation process. They typically need to perform additional steps to fully activate their account.
PASSWORD_EXPIREDUsers with this status typically need to reset their password before they can log in again. Their current password has expired, and they must create a new one to regain access.
SUSPENDEDSuspended users have had their access temporarily revoked. They cannot log in or access resources until their suspension is lifted.
LOCKED_OUTThis status indicates that the user's account has been locked due to multiple failed login attempts. Locked-out users cannot log in until their account is unlocked by an administrator or until a certain amount of time has passed.

Application Setup​

This section explains the process of setting up a service account in Okta for use within the collections process.

Steps

  1. Login to Okta dashboard, https://instance.okta.com/admin/dashboard.
  2. From the main menu, navigate to Security > API. 
  3. Under Tokens tab, click on Create token button.
  4. Provide a suitable name and click Create token.
  5. Copy the token and retain it for use with G&L.
  6. Click OK.
