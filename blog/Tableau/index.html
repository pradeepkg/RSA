Tableau

July 15, 2023 · 59 min read
Pradeep Kadambar
Creator
Tableau is a powerful data visualization tool that transforms complex datasets into interactive and insightful visuals, aiding informed decision-making. Its user-friendly interface enables easy creation of charts, graphs, and dashboards, enhancing data exploration and presentation across various industries and professions.

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ✔️ | Entitlements ❌

In Tableau Cloud, users, groups, and site roles are used to control access to content and features.

Users are individual people who have access to a Tableau Cloud site. Users can be assigned different site roles, which determine what they can do on the site.

Groups are collections of users. Groups can be used to simplify the process of assigning site roles to multiple users.

Site roles define the level of access a user has to a Tableau Cloud site. Site roles include:

  • Viewer: Viewers can view published content, but they cannot create or modify content.
  • Explorer: Explorers can create and modify content, but they cannot publish content to the site.
  • Creator: Creators can create, modify, and publish content to the site.
  • Site Administrator Creator: Site Administrator Creators have all the capabilities of Creators, plus the ability to manage the site's users and settings.
  • Site Administrator Explorer: Site Administrator Explorers have all the capabilities of Explorers, plus the ability to manage the site's users and settings.

Account Collector​

This section explains the process for configuring an account collector for the Tableau application. We will use the RSA G&L Generic REST collector 1 for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Tableau.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details.


    The URL contains the site URL https://prod-useast-b.online.tableau.com (for my instance) which needs to be updated to suit your deployment and the site it. Site ID can be retrieved from the response to the token authentication request.

  6. Click Next. Select the data types that the collector will be collecting from Tableau

  7. Click Next. On the Account Data page, enter the following configurations.




Request
MethodGET
URL/users
Query Parameters
fieldsid,email,lastLogin
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
AccountId$.users.*.*.email
Id$.users.*.*.id
LastLogin$.users.*.*.lastLogin

  1. Click Next. On the Group Data page, enter the following configurations.




Request
MethodGET
URL/groups
Query Parameters
fieldsid,name
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
GroupId$.groups.*.*.name
Id$.groups.*.*.id
  1. Click Next. On the Account Group Membership page, enter the following configurations.




Request
MethodGET
URL/groups/${Id}/users
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
Account Identifier$.users.user.*.id
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account.

💡 Info: This value MUST always be mapped to the Account Name for any generic REST collector.

  1. Click Finish. Use the Test function to make sure the configurations are accurate.

Entitlement Collector​

Role Members​

This section explains the process for configuring an entitlement collector to collect site roles and assignments from the Tableau application. We will use the RSA G&L Generic REST collector for this purpose.

info

Since Tableau does not provide an API for listing site roles, we will use a custom web service for the EDC. Refer to Custom Web Service : Site Roles for more information.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on Tableau.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector. 💡 Note that the Use Base URL for all API(s) will be unchecked.
  6. Click Next. Select the data types that the collector will be collecting from Tableau.

  7. Click Next. On the App Role Data page, enter the following configurations.



Request
MethodGET
URLhttps://instance.securid.com/aveksa/custom/external_jsp/TableauSiteRolesv2.jsp
Request Headers
Acceptapplication/json
Response Mappings
ApplicationRoleId$.roles.*.role
Description$.roles.*.description
  1. Click Next. On the App-Role Account Membership collection page, enter the following configurations.




Request
MethodGET
URLhttps://instance.online.tableau.com/api/3.2/sites/SITE_ID/users
Query Parameters
fieldsid,siteRole
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
Application Role Id$.users.user.*.siteRole
Account Identifier$.users.user.*.id
  1. Click Next. In the Map Collector Attributes to App Role Attributes, map the collected account attributes.

  2. Click Next. In the Account Evaluation, map the account collector to account value.

  3. Click Next and then click Finish. Use the Test function to make sure the configurations are accurate.

Application Setup​

This section explains the process of setting up Personal Access Token (PAT) in Tableau for use within the collections process.

Steps

  1. Login to Tableau site https://instance.online.tableau.com/#/site/SITE/home.
  2. By default, Personal Access Token (PAT) is disabled.
  3. Navigate to Settings in the navigation panel and click on General tab.
  4. Go to Personal Access Tokens section and enable PAT.

  5. Navigate to Users in the navigation panel and click on the user whose PAT needs to be generated.
  6. Click on Settings tab.
  7. Go to Personal Access Tokens section. Provide a suitable token name and click Create Token.

  8. Copy the Secret and click Done.

Authentication​

Token​

This section explains the process of setting up RSA G&L ADC with token authentication.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Create or Edit a generic REST ADC that will be using Token authentication.
  3. On the Connection Details page, configure the following.



Request
MethodPOST
Token URLhttps://YOUR_INSTANCE/api/3.13/auth/signin
UsernameYOUR_PAT_NAME
PasswordYOUR_PAT
Request Headers
Acceptapplication/json
Content-Typeapplication/xml
Request Body
<?xml version="1.0" encoding="UTF-8"?>
<tsRequest>
	<credentials personalAccessTokenName="${TokenUsername}" personalAccessTokenSecret="${TokenPassword}">
		<site contentUrl="rsaps" />
	</credentials>
</tsRequest>
Response Mappings
Response Path$.credentials.token
  1. Save the collector configuration & test.

Custom Web Service​

Site Roles​

Since Tableau does not provide an API for listing site roles, we will use a custom JSP that will publish the site roles.

Steps

  1. Open a text editor and save the contents below as TableauSiteRolesv2.jsp.
<%@ page language="java" contentType="application/json" %>
<%
    String jsonData = "{"
        + "\"roles\": ["
        + "{\"role\":\"Creator\",\"description\":\"Users with this role can create and publish content such as workbooks and data sources on the Tableau site.\"},"
        + "{\"role\":\"Explorer\",\"description\":\"Users assigned as Explorers can interact with and explore content on the Tableau site. They can view and interact with dashboards and reports but do not have the ability to create or publish new content.\"},"
        + "{\"role\":\"ExplorerCanPublish\",\"description\":\"Similar to the \\\"Explorer\\\" role, users with this role can explore content on the site. Additionally, they have the ability to create and publish their own content.\"},"
        + "{\"role\":\"ServerAdministrator\",\"description\":\"Users with this role have full administrative control over the entire Tableau Server. They can manage users, groups, projects, data sources, and server-wide settings.\"},"
        + "{\"role\":\"SiteAdministratorExplorer\",\"description\":\"Site Administrators have full control over a specific site. For the Explorer role, this means they can manage users, content, and site-specific settings.\"},"
        + "{\"role\":\"SiteAdministratorCreator\",\"description\":\"Similar to SiteAdministratorExplorer, Site Administrators with the Creator role can manage users, content, and site settings. They also have the ability to create new content.\"},"
        + "{\"role\":\"Unlicensed\",\"description\":\"Users assigned this role do not have access to content on the Tableau site. It is typically used for users who do not need access to Tableau resources.\"},"
        + "{\"role\":\"ReadOnly\",\"description\":\"Users with this role can view content on the Tableau site but do not have the ability to interact with or modify it. They can consume dashboards and reports without making changes.\"},"
        + "{\"role\":\"Viewer\",\"description\":\"Similar to the ReadOnly role, Viewers can view and interact with content on the Tableau site, but they also have the ability to make changes to views and save personal copies of them.\"}"
        + "]"
        + "}";
%>

<%
    response.setContentType("application/json");
    out.print(jsonData);
%>
  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Admin > User Interface.
  3. Click on Files.
  4. Select External JSP Pages and click on Upload.
  5. Select the TableauSiteRolesv2.jsp and click OK.
  6. Verify by accessing this URL in the browser https://instance.securid.com/aveksa/custom/external_jsp/TableauSiteRolesv2.jsp
