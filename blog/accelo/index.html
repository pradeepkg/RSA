Accelo

December 9, 2023 · 24 min read
Pradeep Kadambar
Creator
Accelo is a cloud-based platform that streamlines business operations through automation and collaboration. It integrates project management, CRM, and service tracking to enhance workflow efficiency. With features like time tracking and invoicing, Accelo aims to optimize project delivery and client engagement for professional service businesses.

Versions: 8.0.0 P1

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ❌ | Entitlements ❌

Application​

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Application.
  3. Click on Create Application and select Other Application.
  4. Enter the details as shown below.
  5. Click Finish.

Account Collector​

This section explains the process for configuring an account collector for the Accelo application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Accelo.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details. This article will use Token based authentication to invoke REST APIs.

  6. Click Next. Select the data types that the collector will be collecting from Accelo

  7. Click Next. On the Account Data page, enter the following configurations.




Request
MethodGET
URL/api/v0/staff
Query Parameters
_fieldsid,email,username,standing
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
AccountId$.response.*.username
Id$.response.*.id
Email$.response.*.email
Status$.response.*.standing
  1. Click Next. On the Group Data page, enter the following configurations.




Request
MethodGET
URL/api/v0/groups
Query Parameters
_fieldstitle,id,standing
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
GroupId$.response.*.title
Id$.response.*.id
Status$.response.*.standing
  1. Click Next. On the Account Group Membership page, enter the following configurations.




Request
MethodGET
URL/api/v0/staff/memberships
Query Parameters
_fieldsgroup_id,staff_id
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
Account Identifier$.response.*.staff_id
Group Identifier$.response.*.group_id
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account attribute.


    💡 Info: This value MUST always be mapped to the Account Name for any generic REST collector.
  6. Click Finish. Use the Test function to make sure the configurations are accurate.

Application Setup​

This section explains the process of setting up token access in Accelo for integration with G&L.

Steps

  1. Login to Accelo console https://instance.accelo.com/.
  2. On the left hand bottom corner, click on Settings.
  3. Under Administration panel, click on API.
  4. Click on Register Application and provide the details as shown.

  5. Click Save. Under API Applications, click on Generate Token.

  6. Provide necessary permissions which in our case is read(all) and click Generate Token.

  7. Note the token value for use in collector configurations.
