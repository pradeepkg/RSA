PingOne

July 22, 2023 · 32 min read
Pradeep Kadambar
Creator
PingOne is an identity and access management (IAM) solution by Ping Identity. It offers secure user authentication, authorization, and single sign-on across applications. With advanced features like multifactor authentication and adaptive security, PingOne ensures streamlined access control while enhancing user experience and safeguarding sensitive data.

Application​

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Application.
  3. Click on Create Application and select Other Application.
  4. Enter the details as shown below.
  5. Click Finish.

Account Collector​

This section explains the process for configuring an account collector for the PingOne application. We will use the RSA G&L Generic REST collector 1 for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on PingOne.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details. PingOne supports both OAuth and API key based authentication. We will token authentication for our collectors in this article. Refer to Token Authentication section.

  6. Click Next. Select the data types that the collector will be collecting from PingOne

  7. Click Next. On the Account Data page, enter the following configurations.

URL/users
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
AccountId$._embedded.users.*.email
Id$._embedded.users.*.id
Status$._embedded.users.*.verifyStatus
IsEnabled$._embedded.users.*.enabled
UserName$._embedded.users.*.username

  1. Click Next. On the Group Data page, enter the following configurations.



Request

Id$._embedded.groups.*.id

  1. Click Next. On the Account Group Membership page, enter the following configurations.

MethodGET
URL/users/${Id}

  1. Click Next. On the Subgroup Membership page, enter the following configurations.



Request
MethodGET
URL/groups/${Id}/memberOfGroups

Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
Subgroup Identifier$._embedded.groupMemberships.*.id
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account attribute.

  6. Click Next. In the Sub-group Resolution Rules, map the group attribute.

  7. Click Next. In the Group Owner Resolution Rules, map the group attribute.

  8. Click Finish. Use the Test function to make sure the configurations are accurate.

Application Setup​

This section explains the process of setting up a service account in PingOne for use within the collections process.

Steps

  1. Login to PingOne console https://console.pingone.com/index.html
  2. Navigate to Connections > Applications.

  3. Click on Add Applications.

  4. Enter a suitable name and description and select application type as Worker.

  5. Click on Save.
  6. Click on the toggle to enable the application. Click on Grant Roles to add the necessary roles for the integration account.

  7. Add Identity Data Read Only Permissions roles and click Save.

  8. Click on Configuration tab and copy the Client ID and Client Secret.

Authentication​

Token​

This section explains the process of setting up RSA G&L ADC with token authentication.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Create or Edit a generic REST ADC that will be using Token authentication.
  3. On the Connection Details page, configure the following.



Request
MethodPOST
Token URLhttps://auth.pingone.com/YOUR_ENV_ID/as/token
UsernameNA
PasswordNA
Request Headers
Acceptapplication/json
Content-Typeapplication/x-www-form-urlencoded
AuthorizationBasic AUTH_STRING

AUTH_STRING - The Base64 encoded string of Client_ID:Client_Secret. The REST API to get the token requires basic authentication to be setup, instead of standard practice of sending the credentials in the POST body.
Request Body
grant_type=client_credentials
Response Mappings
Response Path$.access_token
  1. Save the collector configuration & test.
