Application​

Steps

Account Collector​

This section explains the process for configuring an account collector for the Microsoft Dynamics 365 application. We will use the RSA G&L Generic REST collector 1 for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Microsoft Dynamics 365.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details.



    If authentication type is OAuth2, refer to OAuth2 Configuration section.
    If authentication type is Token, refer to Token Configuration section.

  6. Click Next. Select the data types that the collector will be collecting from Microsoft Dynamics 365

Entitlement Collector​

This section explains the process for configuring an entitlement collector for the Microsoft Dynamics 365 application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on Microsoft Dynamics 365.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector.
  6. Click Next. Select the data types that the collector will be collecting from Microsoft Dynamics 365.

  7. Click Next. On the App Role Data page, enter the following configurations.

Application Setup​

This section explains the setup of credentials for the service account that will be used ADC and EDC.

Steps

  1. Login to Azure portal, Microsoft Azure
  2. Select Azure Active Directory

  3. Note down the Tenant ID. This is required in the RSA G&L collector configuration.

  4. Click on App Registrations.

  5. Click on New Registration.
  6. Provide the details as shown below. Under Redirect URI, provide OAuth callback URL (https://instance.company.com:8443/aveksa/oauth/callback) for RSA G&L.

  7. Click Register.
  8. Note down the Client ID. This is required in the RSA G&L collector configuration.
  9. Click on View API Permissions.

  10. Select Microsoft Graph.

  11. Select Delegated Permissions.

  12. Select the below permissions. Note that this gives full access and hence needs to be tuned down to read only permissions.

  13. Click Add Permissions.
  14. Click on Certificates & Secrets. Click on New Client Secret.

  15. Set the Description and suitable expiration period for the client secret.
  16. Click Add. Note down the Client Secret which will be displayed only once. This is required in the RSA G&L collector configuration.

Token Configuration​

This section explains the process of setting up RSA G&L ADC and EDC with token authentication from Microsoft Graph.

Sample API Response
{
    "token_type": "Bearer",
    "expires_in": "3599",
    "ext_expires_in": "3599",
    "expires_on": "1616882517",
    "not_before": "1616878617",
    "resource": "https://xxxxxxxxxx.cloudax.dynamics.com",
    "access_token": "eyJ0eXAiOi -- TRUNCATED -- CJhbGciOiNENBQT0iLCJhcHBpZCIZLw"
}

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Create or Edit a generic REST ADC or EDC that will be using Token authentication.
  3. On the Connection Details page, configure the following.




Request
MethodPOST
Token URLhttps://login.microsoftonline.com/TENANT_ID/oauth2/token
UsernameClient ID
PasswordClient Secret
Request Headers
Acceptapplication/json
Content-Typeapplication/x-www-form-urlencoded
Request Body

grant_type=client_credentials&client_id=${TokenUsername}&client_secret=${TokenPassword}

Response Mappings
Response Path$.access_token
  1. Save the collector configuration & test.
