Epic EMR

October 8, 2023 · 25 min read
Pradeep Kadambar
Creator
Epic Electronic Medical Records (EMR) solution that is widely used by hospitals. To collect data from this application, we use Epic Clarity.

Epic EMR is a 3NF data warehouse and relational database (Oracle or Microsoft SQL database) designed for database developers. It enables users to create data-intensive reports without modifying the user database, identify misuse of medications, as well as reduce the incidence of medical error. Features include data distribution, label shift management, Peer-to-Peer Evaluations, and BusinessObjects Dashboard.

Account Collector​

This section explains the process for configuring an account collector for the Epic EMR application. We will use the SQL Server database collector for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Epic EMR.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the SQL Server database details. Use the Test Connection to make sure the connection details are accurate. Use the latest Microsoft JDBC Driver for SQL Server if NTLM authentication is required.

  6. Click Next. Select the appropriate data that you intend to collect. For the purpose of illustration we will collect accounts, groups and user to account mapping from Epic EMR.

  7. Click Next. On the Account Data page, enter the following query to collect account information.
SELECT
  EMP.SYSTEM_LOGIN EMP,
  EMP.PROV_ID AS SER,
  EMP.NAME AS displayName,
  EMP.USER_ID userID,
  CASE
    WHEN USER_STATUS_C = 1 THEN 0
    WHEN USER_STATUS_C = 2 THEN 1
    ELSE ''
  END AS isDisabled,
  LAST_PW_UPDATE AS laspPasswordSetDate,
  LAST_ACCS_DATETIME AS lastLoginDate
FROM CLARITY_EMP EMP
WHERE EMP.SYSTEM_LOGIN IS NOT NULL‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍

Entitlement Collector​

This section explains the process for configuring an entitlement collector to collect admin roles from Epic EMR application. We will use the SQL Server database collector for this purpose.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on Epic EMR.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the database connection details as shown in Step 5 of Account Collector.
  6. Click Next. Select the data types that the collector will be collecting from Epic EMR.

  7. Click Next. On the General Column Names, map the attribute names.

  8. Click Next. Under Application Role Data section, use the following query to collect admin roles from Epic EMR.
SELECT DISTINCT
  TEMPLT_DSPLY_TITLE AS templateName,
  USER_ID AS template,
  TEMPLT_OWNER_PRIM_C AS owner
FROM CLARITY_EMP_2
WHERE TEMPLT_DSPLY_TITLE IS NOT NULL‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍‍

The following are the important attributes


9. Click Next. On the Account Data section, enter the query below to collect mappings for account to roles.

SELECT 
  TEMPLATE.LNK_TEMPLT_ID as action, 
  EMP.SYSTEM_LOGIN as account
FROM 
  CLARITY_EMP EMP 
  LEFT OUTER JOIN EMP_AVAIL_TEMPLT TEMPLATE ON TEMPLATE.USER_ID = EMP.USER_ID 
WHERE 
  TEMPLATE.LNK_TEMPLT_ID is not null 
  and EMP.SYSTEM_LOGIN is not null 
UNION 
SELECT 
  SUBTEMPLATE.APPLD_SUBTMPLT_ID as action, 
  EMP.SYSTEM_LOGIN as account 
FROM 
  CLARITY_EMP EMP 
  LEFT OUTER JOIN SUBTEMPLATES SUBTEMPLATE ON SUBTEMPLATE.USER_ID = EMP.USER_ID 
WHERE 
  SUBTEMPLATE.APPLD_SUBTMPLT_ID is not null 
  and EMP.SYSTEM_LOGIN is not null
  1. Click Next. In the User Evaluation, map the account to the ADC.
