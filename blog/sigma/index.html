Sigma

May 15, 2024 · 33 min read
Pradeep Kadambar
Creator
Sigma Business Intelligence offers powerful analytics solutions, empowering businesses with actionable insights. Through intuitive interfaces and robust data integration, Sigma BI transforms complex data into strategic decisions. With a focus on user-friendly tools and real-time analytics, Sigma BI accelerates growth and enhances competitiveness in today's dynamic market landscape.

Versions: 8.0.0 P2

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ❌ | Entitlements ❌

Application​

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Application.
  3. Click on Create Application and select Other Application.
  4. Enter the details as shown below.
  5. Click Finish.

Account Collector​

This section explains the process for configuring an account collector for the Sigma application. We will use the RSA G&L Generic REST collector 📘 for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Sigma.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details.



    Refer to Token Authentication for more information on authentication configuration.

  6. Click Next. Select the data types that the collector will be collecting from Sigma.

  7. Click Next. On the Account Data page, enter the following configurations.



Request
MethodGET
URL/v2/members
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
AccountId$.entries.*.email
Id$.entries.*.memberId
UserKind$.entries.*.userKind
isInactive$.entries.*.isInactive
MemberType$.entries.*.memberType
  1. Click Next. On the Group Data page, enter the following configurations.



Request
MethodGET
URL/v2/teams
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
GroupId$.entries.*.name
Id$.entries.*.teamId
Description$.entries.*.description
  1. Click Next. On the Account Group Membership page, enter the following configurations.



Request
MethodGET
URL/v2/teams/${Id}/members
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
Account Identifier$.entries.*.userId
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account.

  6. Click Finish. Use the Test function to make sure the configurations are accurate.

Authentication​

Token​

This section explains the process of setting up RSA G&L ADC with token authentication.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Create or Edit a generic REST ADC that will be using Token authentication.
  3. On the Connection Details page, configure the following.



Request
MethodPOST
Token URLhttps://aws-api.sigmacomputing.com/v2/auth/token
UsernameClient ID
PasswordClient Secret
Request Headers
Acceptapplication/json
Content-Typeapplication/x-www-form-urlencoded
Request Body
grant_type=client_credentials&client_id=${TokenUsername}&client_secret=${TokenPassword}
Response Mappings
Response Path$.access_token
  1. Save the collector configuration & test.

Application Setup​

This section explains the process of setting up Sigma for integration with G&L.

Steps

  1. Login to Sigma dashboard https://app.sigmacomputing.com/ as administrator.
  2. On the right hand corner, click on the user profile and click on Administration.

  3. Select Developer Access and click on Create new.

  4. Select the REST API option under privileges and provide a suitable name for the integration. Select te appropriate user for the integration owner.

  5. Click on Create and copy the Client ID and Client Secret and retain for use with G&L.
