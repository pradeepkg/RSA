GitLab

October 11, 2023 · 32 min read
Pradeep Kadambar
Creator
GitLab is a DevOps platform that offers a comprehensive suite of tools for software development, version control, and CI/CD. It enables teams to manage repositories, track issues, perform code reviews, and automate the software delivery process. With its self-hosted or cloud-based options, GitLab provides organizations with a flexible and integrated solution to enhance collaboration, streamline workflows, and accelerate software development.

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ❌ | Entitlements ❌

Users Users are individuals who have an account on the GitLab SaaS platform. Each user has a unique username and email address associated with their account. Users can belong to multiple groups and have different roles within those groups or projects.

Groups Groups in GitLab SaaS allow for organizing projects and users. A group can include multiple projects and members, enabling efficient management and collaboration across related repositories. Groups provide a way to apply permissions and access control collectively to a set of projects and users.

Roles GitLab SaaS offers various roles that define the level of access and permissions a user has within a project or group. The common roles include:

Owner The highest level of authority within a project or group. Owners have full control over the settings, members, and permissions.

Maintainer Maintainers have administrative privileges within a project or group. They can manage the project, including repositories, branches, and permissions.

Developer Developers have write access to the repository and can contribute code, create merge requests, and participate in project activities.

Reporter Reporters have read-only access and can view project information, including source code, issues, and merge requests. They cannot make any modifications.

Guest Guests have limited access and can only view basic project information without making any modifications.

Permissions Permissions define what actions a user can perform within a project or group. These actions may include creating and modifying repositories, pushing changes, managing issues and merge requests, modifying project settings, and more. GitLab SaaS allows administrators, maintainers, and owners to configure and customize permissions for each role within the system.

Account Collector​

This section explains the process for configuring an account collector for the GitLab application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on GitLab.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details. The endpoint uses a static token for authentication.

    .
  6. Click Next. Select the data types that the collector will be collecting from GitLab

  7. Click Next. On the Account Data page, enter the following configurations.




Request
MethodGET
URL/api/v4/users
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
AccountId$.*.username
Id$.*.id
State$.*.state
  1. Click Next. On the Group Data page, enter the following configurations.




Request
MethodGET
URL/api/v4/groups
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
GroupId$.*.name
Id$.*.id
Description$.*.description
  1. Click Next. On the Account Group Membership page, enter the following configurations.




Request
MethodGET
URL/api/v4/groups/${Id}/members
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
AccountId$.*.username
  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the collected group attributes.

  3. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  4. Click Next. In the User Resolution Rules, map the user to account.

  5. Click Next. In the Member Account Resolution Rules, map the account attribute.

  6. Click Finish. Use the Test function to make sure the configurations are accurate.

GitLab: Setup​

Personal Access Tokens (PAT)​

This section explains the process of acquiring the authentication data required for collectors from GitLab.

Steps

  1. Login to GitLab profile https://gitlab.com/-/profile
  2. Navigate to Access Tokens
  3. Configure the token details as provided below. The token has a maximum validity of 1 year.

  4. Click Create personal access token.
  5. Copy the token and save it for use within collector configuration.
