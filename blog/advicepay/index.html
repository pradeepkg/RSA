AdvicePay

May 23, 2024 ¬∑ 68 min read
Pradeep Kadambar
Creator
AdvicePay is a secure online payment platform designed specifically for financial advisors. It facilitates fee-for-service transactions, ensuring compliance with industry regulations like ADV and SEC. Advisors can create custom payment schedules, invoice clients, and track payments seamlessly. AdvicePay streamlines billing processes, enabling advisors to focus more on delivering valuable financial guidance to their clients.

Versions: 8.0.0 P2

Overview‚Äã

Admin Roles‚Äã

  • Admin: Can do almost everything the account owner can do, including inviting clients, requesting payments, canceling invoices, and exporting transaction reports. However, Admins cannot view billing bank information, add or delete Admin and Analyst users, change the company name, or delete the account.
  • Analyst: Can only view activity on the account, such as clients, invoices, and transfers, as well as export transaction reports. Analysts cannot view billing bank information or make any changes.
  • Developer: Can read and write, as well as access API keys.

Advisor Roles‚Äã

  • Account Owner: Responsible for managing all firm settings and data, including banking information. Holds complete control over billing clients.
  • Full-Access Advisor: Granted access to view firm data, including clients of other advisors, and can manage billing for their own clients or those of other advisors. They lack authorization to enable eSignatures, alter/view banking details, or delete the account. However, they possess the capability to manage other firm-wide settings.
  • Standard Advisor: Authorized to view and manage billing for their own clients only. They are not permitted to view or control firm-wide settings.
  • Read Only Advisor: Granted login access solely for viewing payment statuses and their own clients' details. Suitable for advisors who need to monitor assigned clients and invoice activity without directly billing them.
  • No Login Advisor: Does not possess login credentials to access account activity. This option is optimal for centralized billing and assigning clients to No-Login Advisors for organizational purposes.

Advisor Permission Types‚Äã

This section explains the process for configuring an entitlement collector to collect advisor permission types and assignments from the AdvicePay application. We will use the RSA G&L Generic REST collector for this purpose.

info

Since AdvicePay does not provide an API for listing advisor permission types, we will use a custom web service for the EDC. Refer to Custom Web Service : Advisor Permission Types for more information.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on AdvicePay.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector. üí° Note that the Use Base URL for all API(s) will be unchecked.
  6. Click Next. Select the data types that the collector will be collecting from AdvicePay.

  7. Click Next. On the App Role Data page, enter the following configurations.



Request
MethodGET
URLhttps://instance.securid.com/aveksa/custom/external_jsp/AdvicePayAdvisorRolesv1.2.jsp
Request Headers
Acceptapplication/json
Response Mappings
ApplicationRoleId$.advisor_roles.*.role
Description$.advisor_roles.*.description
  1. Click Next. On the App-Role Account Membership collection page, enter the following configurations.



Request
MethodGET
URLhttps://app.advicepay.com/api/public/v1/advisors
Request Headers
Acceptapplication/json
AuthorizationBearer ${AuthenticationToken}
Response Mappings
Application Role Id$.items.*.roleTitle
Account Identifier$.items.*.id
  1. Click Next. In the Map Collector Attributes to App Role Attributes, map the collected account attributes.

  2. Click Next. In the Account Evaluation, map the account collector to account value.

  3. Click Next and then click Finish. Use the Test function to make sure the configurations are accurate.

Token‚Äã

This section explains the process of setting up RSA G&L ADC with token authentication.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Create or Edit a generic REST ADC that will be using Token authentication.
  3. On the Connection Details page, configure the following.



Request
MethodPOST
Token URLhttps://app.advicepay.com/oauth2/access_token
UsernameClient ID
PasswordClient Secret
Request Headers
Acceptapplication/json
Content-Typeapplication/x-www-form-urlencoded
Request Body
grant_type=client_credentials&client_id=${TokenUsername}&client_secret=${TokenPassword}
Response Mappings
Response Path$.access_token
  1. Save the collector configuration & test.

Custom Web Service‚Äã

Advisor Permission Types‚Äã

Since AdvicePay does not provide an API for listing advisor permission types, we will use a custom JSP that will publish this as web service.

Steps

  1. Open a text editor and save the contents below as AdvicePayAdvisorRolesv1.2.jsp
<%@ page language="java" contentType="application/json" %>

<%
String jsonData = "{\n" +
    "  \"advisor_roles\": [\n" +
    "    {\n" +
    "      \"role\": \"Managing Advisor\",\n" +
    "      \"description\": \"Can see office-wide data.\"\n" +
    "    },\n" +
    "    {\n" +
    "      \"role\": \"Full-access Advisor\",\n" +
    "      \"description\": \"Can see firm-wide data and update firm settings.\"\n" +
    "    },\n" +
    "    {\n" +
    "      \"role\": \"Advisor\",\n" +
    "      \"description\": \"Gives full access to AdvicePay.\"\n" +
    "    },\n" +
    "    {\n" +
    "      \"role\": \"Read-only Advisor\",\n" +
    "      \"description\": \"Gives login access but no power to invoice clients.\"\n" +
    "    },\n" +
    "    {\n" +
    "      \"role\": \"No-login Advisor\",\n" +
    "      \"description\": \"Gives no login, and is completely managed by home office.\"\n" +
    "    }\n" +
    "  ]\n" +
    "}";
%>

<%
    response.setContentType("application/json");
    out.print(jsonData);
%>
  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Admin > User Interface.
  3. Click on Files.
  4. Select External JSP Pages and click on Upload.
  5. Select the AdvicePayAdvisorRolesv1.2.jsp and click OK.
  6. Verify by accessing this URL in the browser https://instance.securid.com/aveksa/custom/external_jsp/AdvicePayAdvisorRolesv1.2.jsp
