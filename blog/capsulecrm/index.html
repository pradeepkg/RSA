Capsule CRM

December 29, 2023 · 25 min read
Pradeep Kadambar
Creator

Capsule CRM is a user-friendly customer relationship management system designed for small businesses. It streamlines contact management, tracks interactions, and centralizes communication, fostering efficient collaboration. With a clean interface and customizable features, Capsule CRM helps businesses stay organized and build stronger relationships with their clients.

Versions: 8.0.0

Identities ❌ | Accounts ✔️ | Groups ✔️ | App Roles ❌ | Entitlements ❌

Application​

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Application.
  3. Click on Create Application and select Other Application.
  4. Enter the details as shown below.
  5. Click Finish.

Account Collector​

This section explains the process for configuring an account collector for the Capsule CRM application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Capsule CRM.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details. This article will use Token based authentication to invoke REST APIs.

  6. Click Next. Select the data types that the collector will be collecting from Capsule CRM

  7. Click Next. On the Account Data page, enter the following configurations.




    Request
    MethodGET
    URL/api/v2/parties
    Request Headers
    Acceptapplication/json
    AuthorizationBearer ${AuthenticationToken}
    Response Mappings
    AccountId$.parties.*.emailAddresses.*.address
    Id$.parties.*.owner.id
    UserName$.parties.*.owner.username
    IsDeleted$.parties.*.owner.deleted
  8. Click Next. On the Group Data page, enter the following configurations.




    Request
    MethodGET
    URL/api/v2/teams
    Request Headers
    Acceptapplication/json
    AuthorizationBearer ${AuthenticationToken}
    Response Mappings
    GroupId$.teams.*.name
    Id$.teams.*.id
    Description$.teams.*.description
  9. Click Next. On the Account Group Membership page, enter the following configurations.




    Request
    MethodGET
    URL/api/v2/teams
    Query Parameters
    embedmemberships
    Request Headers
    Acceptapplication/json
    AuthorizationBearer ${AuthenticationToken}
    Response Mappings
    Account Identifier$.teams.*.memberships.*.user.id
    Group Identifier$.teams.*.id
  10. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  11. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  12. Click Next. In the Map Collector Attributes to Group Attributes, map the collected group attributes.

  13. Click Next. In the User Resolution Rules, map the user to account.

  14. Click Next. In the Member Account Resolution Rules, map the account attribute.


    💡 Info: This value MUST always be mapped to the Account Name for any generic REST collector.
  15. Click Finish. Use the Test function to make sure the configurations are accurate.

Application Setup​

This section explains the process of setting up token access in Capsule CRM for integration with G&L.

Steps

  1. Login to Capsule CRM console https://instance.capsulecrm.com/.
  2. On the right hand top corner, click on profile and select My Preferences.

  3. Click on API Authentication Tokens.

  4. Click on Generate First API Token or Generate New API Token option.
  5. Provide a suitable description and select the Read information from your Capsule account scope for the token. Click Generate Token.

  6. Note the token value for use in collector configurations.
