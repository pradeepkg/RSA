Sumo Logic

October 14, 2023 · 34 min read
Pradeep Kadambar
Author
Sumo Logic is a cloud-based log management and analytics platform that empowers organizations to collect, analyze, and derive insights from their machine data in real time. It offers a unified view of data across applications and infrastructure, enabling users to troubleshoot issues, monitor performance, detect security threats, and drive data-driven decision-making for enhanced operational efficiency and security.

Identities ❌ | Accounts ✔️ | Groups ❌ | App Roles ✔️ | Entitlements ❌

Users in Sumo Logic are individual accounts that interact with the Sumo Logic platform. Users can be individuals within an organization, such as analysts, administrators, developers, and more. Each user is uniquely identified by their email address and has their own set of credentials (username and password) to log in to the Sumo Logic platform.

Roles define a set of permissions and access rights that determine what actions a user can perform within the Sumo Logic platform. Roles are used to control access to various features, functionalities, and data. Sumo Logic offers predefined roles, but you can also create custom roles to fit your organization's specific needs.

Roles can be categorized into different types, such as:

  • Admin Roles: These roles have extensive permissions, allowing users to manage and configure various aspects of the Sumo Logic account, such as data sources, collectors, and dashboards.
  • User Roles: These roles provide more limited permissions, focusing on data analysis, querying, and visualization without the ability to modify account settings.

The relationship between users and roles is established through role assignments. Each user can be assigned one or more roles, and these roles collectively define the user's permissions and access levels within the Sumo Logic platform.

Application​

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Application.
  3. Click on Create Application and select Other Application.
  4. Enter the details as shown below.
  5. Click Finish.

Account Collector​

This section explains the process for configuring an account collector for the Sumo Logic application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as System Administrator.
  2. Navigate to Resources > Applications.
  3. Click on Sumo Logic.
  4. Navigate to Collectors tab and click on Create Account Collector. Enter the details as shown below.

  5. Click Next and enter the connection details.

    .
  6. Click Next. Select the data types that the collector will be collecting from Sumo Logic

  7. Click Next. On the Account Data page, enter the following configurations.




Request
MethodGET
URL/users
Request Headers
Acceptapplication/json
Response Mappings
AccountId$.data.*.email
Id$.data.*.id
IsActive$.data.*.isActive
IsLocked$.data.*.isLocked
LastLogin$.data.*.lastLoginTimestamp

  1. Click Next. In the Map Collector Attributes to Account Attributes, map the collected account attributes.

  2. Click Next. In the Map Collector Attributes to Account Mapping Attributes, map the user reference attribute.

  3. Click Next. In the User Resolution Rules, map the user to account.

  4. Click Finish. Use the Test function to make sure the configurations are accurate.

Entitlement Collector​

This section explains the process for configuring an entitlement collector to collect user roles and assignments from the Sumo Logic application. We will use the RSA G&L Generic REST collector for this purpose.

Steps

  1. Login to RSA G&L console as AveksaAdmin.
  2. Navigate to Resources > Applications.
  3. Click on Sumo Logic.
  4. Navigate to Collectors tab and click on Create Entitlement Collector. Enter the details as shown below.

  5. Click Next and enter the connection details as shown in Step 5 of Account Collector.
  6. Click Next. Select the data types that the collector will be collecting from Sumo Logic.

  7. Click Next. On the App Role Data page, enter the following configurations.

  1. Click Next. In the Map Collector Attributes to App Role Attributes, map the collected account attributes.

  2. Click Next. In the Account Evaluation, map the account collector to account value.

  3. Click Next and then click Finish. Use the Test function to make sure the configurations are accurate

Application Setup​

This section explains the process of setting up Sumo Logic for integration with G&L. Sumo Logic API calls require application key and API key for most invocations.

Steps

  1. Login to Sumo Logic UI, https://service.sumologic.com/.
  2. Under Administration, click on Security.
  3. Click on Access Keys tab.
  4. Click on Add Access Key
  5. Provide a suitable key name and click Save
  6. Copy the Access ID (user name) and Access Key (Password) to be used in Basic Authentication in the collectors. Click Done.
  7. Click on Policies tab and set the Access Keys Deactivation to a longer period. It is 30 days by default.
