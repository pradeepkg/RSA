Active Directory

March 22, 2024 ¬∑ 48 min read
Pradeep Kadambar
Creator

Active Directory (AD) is a Microsoft directory service that manages network resources efficiently. It centralizes authentication, authorization, and network management tasks, providing a unified environment for user and computer management. AD simplifies IT administration, enhances security through access controls, and facilitates seamless communication across a network, making it a crucial component for enterprise environments.

Additional Use Cases

üü† Dynamic DN‚Äã

During the account creation the account can be placed in specific Organizational Units (OU) dynamically based on one or more attributes from the HR sources or key passed through account template.

Examples

  • Place users in OUs based on country code from HR system.
  • Place users in OUs based on the department and office location attributes collected from HCM.

Solution

  1. Add DN Suffix Mappings in the Settings tab.
  2. Add AFX_DN_Suffix_Key1 to the Create Account verb.
  3. Read the distinguishedName output parameter in the Create Account verb.

Items marked by 1Ô∏è‚É£ in this blog are related to this use case.

üü† Change Password on First Logon‚Äã

When creating a new account, it may be necessary for users to change their password during their initial login. This can be accomplished by configuring specific input parameters within the Create Account verb.

The input parameters required are as follows:

ParameterDescription
_AFX_FORCE_REGISTER_USER_CHANGE_PWD- When it is set to 0, the pwdLastSet attribute of the new account will be set to the date/time when the account is created (i.e. the user does not have to set the password at the next logon).
- When it is set to 1, the pwdLastSet attribute of the new account will be set to 0 (i.e. the user must set the password at the next logon).
pwdLastSetSetting to 0 along with userAccountControl to NORMAL_ACCOUNT,PASSWORD_EXPIRED will set the User must change password at next logon on the Active Directory account. This attribute stores the value of the date and time when the user's password was last changed in Windows NT.

This is an alternative solution to setting _AFX_FORCE_REGISTER_USER_CHANGE_PWD
